## 一、项目概述

### 1.1 项目背景
"苍蝇小馆"是一款面向广大吃货的小程序，用于发现、记录和分享那些藏在街头巷尾的土菜馆——它们可能门面不起眼、环境简陋，但味道地道、价格亲民。

### 1.2 产品定位
- 轻量级：无需注册即可浏览，使用门槛低
- UGC 驱动：内容由用户贡献，形成吃货社区
- 实用导向：聚焦"好不好吃"，而非"好不好看"

### 1.3 目标用户
- 美食爱好者：寻找身边的隐藏美味
- 探店达人：乐于分享发现的宝藏小店
- 本地居民：推荐自家楼下的好味道

## 二、核心功能

### 2.1 用户角色

| 角色 | 权限 |
|------|------|
| 游客 | 浏览馆子列表、查看详情、搜索筛选 |
| 注册用户 | 以上 + 录入馆子、收藏、评论、点赞 |
| 管理员 | 内容审核、用户管理、数据统计 |

### 2.2 功能模块

#### 模块一：馆子浏览

| 功能 | 描述 |
|------|------|
| 首页推荐 | 按热度/距离/最新排序展示馆子列表 |
| 搜索 | 支持按店名、地点、菜系搜索 |
| 筛选 | 按区域、人均价格、菜系类型筛选 |
| 地图模式 | 在地图上查看附近馆子分布 |

#### 模块二：馆子详情

| 功能 | 描述 |
|------|------|
| 基本信息 | 店名、地址、电话、营业时间、人均消费 |
| 菜品推荐 | 招牌菜、必点菜（用户贡献） |
| 用户评价 | 评论列表、评分（味道/分量/性价比） |
| 图片展示 | 用户上传的环境、菜品照片 |
| 地图导航 | 一键跳转地图导航 |

#### 模块三：馆子录入

| 功能 | 描述 |
|------|------|
| 基本信息 | 店名、地址（支持地图选点）、电话、营业时间 |
| 菜系标签 | 如：川菜、湘菜、粤菜、东北菜等 |
| 人均价格 | 手动输入或选择价格区间 |
| 推荐菜品 | 录入招牌菜名称 |
| 图片上传 | 上传环境或菜品照片（可选） |
| 个人评价 | 填写首次评价内容 |

#### 模块四：用户中心

| 功能 | 描述 |
|------|------|
| 我的收藏 | 收藏的馆子列表 |
| 我的贡献 | 录入的馆子、发表的评论 |
| 个人信息 | 昵称、头像、简介 |

#### 模块五：互动功能

| 功能 | 描述 |
|------|------|
| 收藏 | 收藏感兴趣的馆子 |
| 点赞 | 为馆子或评论点赞 |
| 评论 | 发表评价，支持图片 |

## 三、数据模型（初步）

### 3.1 馆子（Restaurant）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| name | String | 店名 |
| address | String | 详细地址 |
| longitude | Double | 经度 |
| latitude | Double | 纬度 |
| phone | String | 联系电话 |
| cuisine_type | String | 菜系类型 |
| avg_price | Integer | 人均消费（元）|
| business_hours | String | 营业时间 |
| rating | Double | 综合评分 |
| create_user_id | Long | 录入用户 |
| create_time | DateTime | 创建时间 |
| status | Integer | 状态（正常/待审核/已下架）|

### 3.2 菜品（Dish）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| restaurant_id | Long | 所属馆子 |
| name | String | 菜品名称 |
| recommend_count | Integer | 推荐次数 |
| status | Integer | 状态 |

### 3.3 评论（Review）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| restaurant_id | Long | 所属馆子 |
| user_id | Long | 评论用户 |
| content | String | 评论内容 |
| taste_score | Integer | 味道评分（1-5）|
| portion_score | Integer | 分量评分（1-5）|
| value_score | Integer | 性价比评分（1-5）|
| images | String | 图片URL（JSON数组）|
| create_time | DateTime | 创建时间 |

### 3.4 用户（User）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| openid | String | 微信openid |
| nickname | String | 昵称 |
| avatar | String | 头像URL |
| create_time | DateTime | 创建时间 |

### 3.5 收藏（Favorite）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| user_id | Long | 用户ID |
| restaurant_id | Long | 馆子ID |
| create_time | DateTime | 创建时间 |

## 四、技术方案建议

### 4.1 前端
- 平台：微信小程序
- 框架：uni-app（可扩展至其他小程序平台）
- 地图：腾讯地图 SDK

### 4.2 后端
- 语言：Java 8
- 框架：Spring Boot 2.7.x
- 数据库：MySQL 5.7
- 缓存：Redis
- 对象存储：腾讯云 COS
- 微信小程序：weixin-java-miniapp 或其他合适框架

### 4.3 管理后台（PC端）

| 项目 | 技术选型 |
|------|----------|
| 前端框架 | Vue 2.x |
| UI 组件库 | Element UI |
| 后端 | 与小程序共用同一套后端服务 |

#### 管理后台功能
- 馆子审核（待审核列表、审核通过/驳回）
- 内容管理（馆子、评论的增删改查）
- 用户管理（用户列表、状态管理）
- 数据统计（录入量、活跃度等）

---

### 4.4 部署
- 部署方式：现有服务器，Docker 容器化部署

## 七、开发计划
